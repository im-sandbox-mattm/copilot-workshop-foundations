# Copilot Instructions

- Keep changes small and easy to explain live.
- Follow existing Java Spring Boot and React TypeScript patterns in this repo.
- Reuse existing DTO, service, and styling conventions before creating new ones.
- Do not change backend contracts for frontend-only tasks unless the prompt requires it.
- Avoid new dependencies unless they remove clear workshop pain.
- Prefer focused edits over broad rewrites.
- For reviews, rank findings as Security, Reliability, Maintainability, then Best Practices.
- For implementations, name the files you plan to touch before editing when the scope spans multiple files.
